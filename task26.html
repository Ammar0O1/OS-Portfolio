The exit() System Call

What is exit()?

exit() is a C library function that terminates the calling process. When a program is done with its work (or encounters an error it can't recover from), it calls exit() to shut down cleanly.

It performs cleanup operations — like flushing output buffers and calling registered cleanup handlers — then tells the kernel to remove the process.

Key point: exit() takes an integer argument called the exit status. By convention, 0 means success and any non-zero value means an error occurred.

Syntax

#include <stdlib.h>

void exit(int status);

Header: <stdlib.h>
Parameter: status — integer sent to the parent process (only bits 0–255 are used).
Returns: Nothing — this function never returns.

Code Example

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // Child process
        printf("Child: doing work...\n");
        exit(0);  // success
    }

    // Parent waits for child
    int status;
    wait(&status);

    if (WIFEXITED(status))
        printf("Child exited with %d\n", WEXITSTATUS(status));

    return 0;
}

What Happens When exit() Is Called

  1. Calls all cleanup functions registered with atexit() in reverse order
  2. Flushes all open output buffers (e.g. stdout)
  3. Closes all open file streams
  4. Removes temporary files created by tmpfile()
  5. Calls the _exit() system call, which hands control to the kernel
  6. Kernel frees memory, closes file descriptors, and sends SIGCHLD to parent

Common Exit Codes

CodeMeaning
0Success (EXIT_SUCCESS)
1General error (EXIT_FAILURE)
2Misuse of command / bad arguments
126Command found but not executable
127Command not found
130Terminated by Ctrl+C (SIGINT)
137Killed with kill -9 (SIGKILL)
