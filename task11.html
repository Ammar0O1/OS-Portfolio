How devices move data without burdening the CPU

What is DMA?

The Core Idea

Direct Memory Access (DMA) is a hardware feature that allows certain devices — such as disk controllers, network cards, and sound cards — to transfer data directly to and from main memory without requiring the CPU to handle every byte of the transfer.

A dedicated chip called the DMA Controller (DMAC) manages the transfer on behalf of the device. The CPU only needs to set up the transfer and is then free to do other work while the data moves. When the transfer is complete, the DMA controller raises an interrupt to notify the CPU.

DMA is fundamental to high-performance I/O. Without it, reading a large file from disk would force the CPU to sit in a busy loop copying bytes — wasting processing time that could be used for computation.

Without DMA vs. With DMA

Without DMA (Programmed I/O)

  • CPU requests data from device
  • CPU polls or waits for each byte/word
  • CPU copies every byte to memory manually
  • CPU is 100% occupied during transfer
  • No other process can use CPU while transfer runs
  • Wastes CPU cycles on trivial data movement

With DMA

  • CPU tells DMAC: source, destination, size
  • DMA Controller takes over the bus
  • Data moves directly: Device ↔ Memory
  • CPU is free to execute other processes
  • Transfer runs in parallel with CPU work
  • Interrupt signals CPU when transfer is done

How DMA Works — Step by Step

  1. The CPU identifies that an I/O transfer is needed (e.g., reading a file from disk).
  2. The CPU programs the DMA Controller with: the source address (device buffer), the destination address (memory location), the transfer size (number of bytes), and the direction (read or write).
  3. The CPU issues a "start transfer" command to the DMAC and then continues executing other instructions.
  4. The DMA Controller requests control of the system bus. The CPU grants bus access (called cycle stealing or burst mode).
  5. The DMAC transfers data directly between the device and memory, byte by byte or in blocks, without CPU involvement.
  6. Once the transfer is complete, the DMA Controller raises a hardware interrupt to the CPU.
  7. The CPU handles the interrupt, acknowledges the transfer is done, and continues its work with the data now ready in memory.

DMA Transfer Modes

How the DMA Controller Uses the Bus

Mode Description CPU Impact
Burst Mode DMAC holds the bus for the entire transfer duration, transferring all data in one continuous block. CPU cannot use the bus during the burst. High throughput but CPU is temporarily bus-locked.
Cycle Stealing DMAC transfers one word at a time, temporarily "stealing" a single bus cycle, then releases it. CPU experiences small periodic delays. Smooth interleaving of CPU and DMA activity.
Transparent Mode DMAC only uses the bus when the CPU is not using it (CPU is executing instructions that don't need the bus). No CPU slowdown at all. Slowest DMA mode but completely non-intrusive.

Where DMA Is Used

Device / Use CaseWhy DMA Matters
💾 Disk / SSD ControllersTransfers large blocks of file data to RAM without tying up the CPU.
🌐 Network Interface CardsMoves incoming/outgoing packet data directly into kernel buffers.
🔊 Sound CardsStreams audio data to/from memory in real-time for continuous playback.
📷 GPUs & VideoRapidly copies large frame buffers and textures between memory regions.
📡 USB ControllersHandles bulk USB transfers to memory with minimal CPU involvement.
