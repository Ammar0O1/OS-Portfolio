What is this demo?

A process is a program in execution. While a program is passive code stored on disk, a process is active because it is loaded into memory and managed by the operating system. During its lifetime, a process moves through different states such as New, Ready, Running, Waiting, and Terminated.

This page demonstrates those states visually. Use the buttons to move process P1 from one state to another, just like the operating system does when it admits a process, dispatches it to the CPU, handles I/O waiting, receives an interrupt, or terminates the process.

CPU / I-O Burst Idea

A process often alternates between CPU burst and I/O burst. When it waits for I/O, the OS can use the CPU for another process instead of leaving it idle.

New

The process is being created. The OS prepares the process and its required information.

Ready

The process is ready to run, but it is waiting for the CPU scheduler to choose it.

Running

The CPU is executing the process instructions. On a single-core system, only one process runs at a time.

Waiting

The process cannot continue until an event occurs, such as I/O completion or receiving a signal.

Terminated

The process has finished execution normally or abnormally and is removed from active scheduling.
