What is read()?

In modern POSIX compliant operating systems, a program that needs to access data from a file stored in a file system uses the read() system call. The file is identified by a file descriptor that is normally obtained from a previous call to open. This system call reads in data in bytes, the number of which is specified by the caller, from the file and stores them into a buffer supplied by the calling process.

In simple words, read() moves data from an input source into the program.

Basic Syntax in C

ssize_t read(int fd, void *buffer, size_t count);
  • fd: File descriptor. For example, 0 means keyboard input.
  • buffer: Memory location where the data will be stored.
  • count: Maximum number of bytes to read.
  • return value: Number of bytes actually read, 0 for end of file, or -1 if an error occurs.

Example C Program

#include <stdio.h> #include <unistd.h> int main() { char buffer[50]; int bytesRead; printf("Enter text: "); bytesRead = read(0, buffer, 50); printf("You entered: "); write(1, buffer, bytesRead); return 0; }

In this example, read(0, buffer, 50) reads up to 50 bytes from the keyboard and stores them in the buffer.

Why is read() Important?

  • It allows programs to receive input from files and devices.
  • It is a low-level system call used by many higher-level functions.
  • It shows how user programs communicate with the operating system kernel.
  • It is commonly used with other system calls such as open(), write(), and close().
