Overview

CPU scheduling is basically how the operating system decides which process should use the CPU next. Since many processes can be ready at the same time, the CPU needs a fair and organized way to choose between them. The main point is to keep the CPU working instead of leaving it idle while processes are waiting.

CPU Scheduler

Chooses one ready process and gives it a turn to run on the CPU.

Ready Queue

Stores the processes that are ready, but still waiting for their CPU turn.

Dispatcher

Does the actual switch when the CPU moves from one process to another.

What is SJF Scheduling?

SJF means Shortest Job First. The idea is easy to understand: the process with the shortest CPU burst gets selected first. So instead of making small jobs wait behind long jobs, the system gives shorter jobs a chance to finish earlier.

Important rule: If two processes have the same burst time, the one that arrived first will run first. So in that case, it works like FCFS as a tie breaker.

Types of SJF

Non-Preemptive SJF

When a process starts running, it stays on the CPU until it finishes. A new shorter process cannot interrupt it.

Preemptive SJF / SRTF

If a new process arrives and has a shorter remaining time, the running process can be paused and replaced.

Scheduling Criteria

These are the main formulas we usually use when solving CPU scheduling questions:

Completion Time = time when process finishes
Turnaround Time = Completion Time - Arrival Time
Waiting Time = Turnaround Time - Burst Time

In preemptive SJF, P1 starts first because it is the only process available at time 0. When P2 arrives at time 1, it has a shorter burst than the remaining time of P1, so P1 is stopped for now. After that, the scheduler keeps choosing the process with the shortest remaining time.

This example shows the idea clearly: in preemptive scheduling, a process does not always finish in one turn. It can be paused if a shorter job arrives.

Summary

  • SJF runs the process with the shortest CPU burst first.
  • It is useful because short jobs do not have to wait too long.
  • In non-preemptive SJF, the running process is not interrupted.
  • In preemptive SJF, the scheduler can stop a process if a shorter one arrives.
  • The weakness is that the exact next CPU burst is not always known, so the OS may need to estimate it.
