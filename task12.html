Introduction

When programmers write code, the computer cannot run it immediately. The code goes through multiple stages before becoming a running application. Two important parts of this process are the Linker and the Loader.

The linker combines object files and libraries into one executable file, while the loader places the executable into memory and starts execution.

What Does the Linker Do?

The linker combines all object files into one executable program. It also connects library functions such as printf() and sqrt().

Main Tasks of the Linker

  • Combines multiple object files
  • Connects external libraries
  • Resolves missing references
  • Creates the final executable file

Example:

main.o + math.o + stdio library → app.exe

What Does the Loader Do?

After the executable file is created, the operating system uses the loader to place the program into RAM and start execution.

Main Tasks of the Loader

  • Loads the executable into memory
  • Allocates memory for the program
  • Loads required libraries
  • Starts execution from main()

Real Life Analogy

Imagine building a car:

  • Compiler: Creates car parts
  • Linker: Assembles the parts into one car
  • Loader: Places fuel in the car and starts the engine
