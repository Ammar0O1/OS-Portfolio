What is wait()?

The wait() system call is used by a parent process to pause its execution until one of its child processes finishes. It helps the parent collect the child process exit status and prevents zombie processes.

Why is it important?

  • It keeps parent and child processes synchronized.
  • It allows the parent to know when the child has ended.
  • It collects the child exit status.
  • It removes zombie processes from the process table.

Simple Meaning

Think of a parent process giving a task to a child process. The parent then says: “I will wait here until my child finishes.” That is exactly what wait() does.

How wait() Works

  1. The parent process creates a child process using fork().
  2. The parent calls wait().
  3. The parent becomes blocked and stops temporarily.
  4. The child process finishes using exit().
  5. The parent collects the child's exit status and continues running.

Syntax

pid_t wait(int *status);

The function returns the process ID of the child that finished. The status variable stores information about how the child process ended.

C Example of wait()

This simple C code shows a parent process waiting for its child process to finish.

#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid;
    int status;

    pid = fork();

    if (pid == 0) {
        // Child process
        printf("Child process is running.\n");
        printf("Child process finished.\n");
    } 
    else {
        // Parent process
        wait(&status);
        printf("Parent waited for child to finish.\n");
    }

    return 0;
}

Important Terms

Term Meaning
Parent Process The process that creates another process.
Child Process The new process created by the parent process using fork().
Blocked State A state where the parent waits and does not continue until the child finishes.
Exit Status A value that tells the parent how the child process ended.
Zombie Process A finished child process whose status has not yet been collected by the parent.

Conclusion

The wait() system call is an important part of process management in Operating Systems. It allows the parent process to pause until the child process finishes, collect the child's exit status, and prevent zombie processes. In simple words, wait() helps the operating system keep parent and child processes organized.
