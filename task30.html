The write() System Call

← Back to Home

What is write()?

write() is a system call from the POSIX standard, available through #include <unistd.h>. It writes a number of bytes from a buffer in your program directly to a file descriptor — which could be the terminal screen, a file, or any other I/O resource.

Unlike printf(), which is a C library function that buffers output, write() goes directly to the operating system kernel with no buffering in between.

Parameters

fd
int

The file descriptor to write to. 0 = stdin, 1 = stdout, 2 = stderr.

buf
void *

Pointer to the data you want to write.

count
size_t

How many bytes to write from the buffer.

File Descriptors

Every process starts with three file descriptors already open:

fdNameWhere it goes
0stdinKeyboard input
1stdoutTerminal screen — most common with write()
2stderrTerminal screen (for errors)
3+filesFiles opened with open() get fd starting from 3

Return Value

write() returns a ssize_t — a signed integer:

Success
> 0

Number of bytes actually written.

Error
-1

Something went wrong. Check errno for the reason.

Code Examples

The count parameter controls exactly how many bytes get written:

count == strlen

int main() {
  write(1, "Software", 8);

n is 9
Warning: When count is greater than the string length, write() reads beyond the buffer and prints garbage characters. Always make sure count matches the actual data size.

Writing to Files

You can use open() to get a file descriptor, then pass it to write() to write to a file instead of the terminal:

#include <unistd.h>
#include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>
int main() {
    int fd;
    fd = open("output.txt", O_CREAT | O_WRONLY, 0642);
    write(fd, "Hello from write()\n", 19);
}
#include <unistd.h>
#include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>
int main() {
    int fd;
    fd = open("output.txt", O_WRONLY | O_APPEND);
    write(fd, "Appended line\n", 14);
}
