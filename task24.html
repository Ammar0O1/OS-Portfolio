What is fork()?

The fork() system call creates a new process by duplicating the calling process. The new process is called the child process, and the calling process is the parent process.

After a successful fork():

  • Parent and child have separate memory spaces
  • Both continue execution from the fork() call
  • They have different Process IDs (PIDs)
  • Child inherits parent's resources and file descriptors
// Simple fork() example #include <unistd.h> #include <stdio.h> int main() { pid_t pid = fork(); if (pid == -1) { perror("fork failed"); } else if (pid == 0) { // Child process printf("Child: PID=%d\n", getpid()); } else { // Parent process printf("Parent: PID=%d, Child PID=%d\n", getpid(), pid); } return 0; }

Return Value

The fork() call returns different values to parent and child:

Return Value Meaning
> 0 Parent process receives child's PID
0 Child process receives 0
-1 Fork failed (no new process created)
💡 Key Point

After fork(), both processes execute the same code but have separate memory spaces. The return value is how they know whether they're parent or child.

Why Use fork()?

  • Create new processes concurrently
  • Server handling multiple clients
  • Process isolation and resource management
  • Foundation for process-based multitasking

Process Tree

How fork() Works (Step by Step)

1 Before fork(): Only parent process exists with its memory, open files, and registers.
2 fork() called: OS creates new process with duplicate of parent's memory and resources.
3 Process Control Block (PCB): OS allocates new PCB for child with unique PID.
4 Both resume: Parent receives child's PID; child receives 0. Both continue from fork() call.
5 Independent execution: Parent and child can execute different code paths based on return value.

Common Fork() Patterns

Pattern 1: Server Handling Multiple Clients

while (1) { // Accept client connection client_socket = accept(server_socket, ...); if (fork() == 0) { // Child: handle this client handle_client(client_socket); exit(0); } // Parent: continue accepting connections }

Pattern 2: Parallel Task Execution

for (int i = 0; i < num_tasks; i++) { if (fork() == 0) { // Child: execute task execute_task(i); exit(0); } } // Parent: wait for all children while (wait(NULL) > 0);
