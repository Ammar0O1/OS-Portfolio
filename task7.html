Process Control Block (PCB)

Every process running on your computer has a kind of "ID card" that the operating system uses to keep track of it. This is called the PCB.

What is a PCB?

You can think of it like a patient file in a hospital. The hospital staff don't memorize your medical history, they just check your file. It has your name, condition, medications, doctor, and room number.

The PCB does the same thing for a process. It is a data record that holds all the important information the OS needs to manage a process.

The PCBs are stored in the ready queue (not the actual code of the process). When the CPU scheduler decides which process runs next, it is really looking at PCBs.

Why is the PCB important?

1. Context Switching: When the OS switches from one process to another, it saves the current process state into its PCB, then loads the next one from its PCB. Without the PCB, the OS would lose track of where a process stopped.

2. Scheduling: The CPU scheduler checks the PCBs in the ready queue to pick which process runs next. The priority and scheduling info are stored there.

3. Threads: Modern operating systems support multiple threads in one process. The PCB is expanded to also track each thread.

Memory info in the PCB

The memory part of the PCB looks different depending on what memory system the OS uses:

Component Used in What it tells the OS
Base / Limit Contiguous Allocation Where the process starts and how big it is
Page Table Paging How to translate a virtual address to a physical one
Segment Table Segmentation Where the code, stack, and data sections are located
