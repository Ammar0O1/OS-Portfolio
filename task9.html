Interprocess Communication (IPC)

How cooperating processes share data. Two main models: Shared Memory and Message Passing.

What is IPC?

A process is independent if it doesn't share data with others. A process is cooperating if it can affect or be affected by other processes. Cooperating processes need IPC.

Reasons for cooperation:

  • Information sharing — multiple apps may need the same data
  • Computation speedup — split a task across multiple cores
  • Modularity — divide the system into separate processes

The Two IPC Models

Shared Memory — processes establish a common memory region and read/write to it directly. Fast, but they must synchronise manually to avoid race conditions.
Message Passing — processes exchange messages via send() and receive() calls handled by the kernel. Safer and works across a network.

Shared Memory vs Message Passing

PropertyShared MemoryMessage Passing
SpeedFast — no kernel per transferSlower — kernel every call
SynchronisationManual (race condition risk)Built-in
ScopeSame machine onlySame machine or network
Best forHigh-throughput local tasksDistributed systems

Blocking vs Non-Blocking

TypeBehaviour
Blocking SendSender halts until message is received
Non-Blocking SendSender continues immediately
Blocking ReceiveReceiver waits until a message arrives
Non-Blocking ReceiveReceiver gets message or null right away
