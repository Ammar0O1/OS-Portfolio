Task 03

Priority Scheduling
(Preemptive)

A CPU scheduling algorithm where each process is assigned a priority, and the CPU always runs the highest-priority process available — even if it means interrupting the one currently running.

How It Works

mechanism

Preemptive Priority

A priority number is associated with each process. The CPU is allocated to the process with the highest priority. If a new process arrives with a higher priority than the currently running one, the OS forcefully pauses the running process and hands the CPU to the new arrival.

analogy

The Moderated Debate

Imagine a moderator giving each speaker a turn. But if breaking news arrives (higher priority), the moderator cuts the current speaker's microphone, saves their spot, and hands the mic to the urgent speaker. When done, the previous speaker resumes.

warning

Starvation

A major problem: low-priority processes may wait indefinitely if higher-priority processes keep arriving. A process that is ready to run but perpetually waiting for the CPU is considered blocked or starved.

solution

Aging

The fix for starvation: gradually increase the priority of processes that wait a long time. For example, increase priority by 1 every 15 time units. Eventually, even the lowest-priority process will become the highest and get its turn.
