1. What is Virtual Memory?

Virtual Memory is a memory management technique used by operating systems to give processes the illusion that they have access to a large continuous memory space, even if the physical RAM is limited.

Main Idea

Programs use virtual addresses instead of physical addresses. The operating system maps virtual memory to physical memory using a structure called the Page Table.

Why Virtual Memory?

  • Allows multitasking
  • Programs can be larger than RAM
  • Improves memory protection
  • Reduces fragmentation
  • Improves system efficiency

Real Life Example

Think of RAM as a desk and Hard Disk as a storage room. Virtual memory lets the operating system move unused data from the desk to storage and bring it back when needed.

2. Paging Concept

Paging divides memory into fixed-size blocks.

Concept Description
Page A fixed-size block of virtual memory
Frame A fixed-size block in physical memory (RAM)
Page Table Maps pages to frames
Page Fault Occurs when a required page is not in RAM

4. Page Fault Explanation

What is a Page Fault?

A Page Fault occurs when a program tries to access a page that is not currently stored in RAM.

What Happens?

  1. CPU requests a page
  2. OS checks the page table
  3. Page not found in RAM
  4. OS loads page from disk
  5. Page table gets updated
  6. Program continues execution

Problem

Accessing the hard disk is much slower than RAM. Too many page faults can reduce system performance.

5. Address Translation Example

The operating system converts a Virtual Address into a Physical Address.

Virtual Page Mapped Frame
0 2
1 1
2 3
3 0

Example Calculation

Suppose the CPU wants Virtual Address: Page 2, Offset 100

From the Page Table: Page 2 → Frame 3

Physical Address becomes: Frame 3 + Offset 100

7. Advantages and Disadvantages

Advantages

  • Efficient memory usage
  • Supports multitasking
  • Programs can exceed RAM size
  • Eliminates external fragmentation
  • Improves security and isolation

Disadvantages

  • Page faults reduce performance
  • Requires extra hardware support
  • Page tables consume memory
  • Disk access is slow
